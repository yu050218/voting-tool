# 万能投票工具开发日志

## 2026-07-10 初步开发

### 1. 需求阅读与范围确认

- 阅读《万能投票工具项目设计报告(1).pdf》的文字内容，重点确认三类核心场景：校内选举、民主测评、作品投票。
- 对照报告中的原型图，确认页面范围包括：平台前台首页、任务登录页、后台任务管理页、新建/编辑任务页、题目与对象配置页、用户投票/测评页、数据统计页。
- 将短周期专业实习的交付目标定位为“可运行、可演示、便于后续接后端”的响应式 Web App。

### 2. 技术选型

- 使用 Vite + React 实现单页应用，启动快、代码结构清晰，适合课程项目和后续交接。
- 使用浏览器 localStorage 保存任务、对象、账号和投票记录，当前版本不依赖外部数据库，便于课堂演示。
- 使用 lucide-react 提供统一图标，减少手写图标和视觉不一致问题。
- 文件命名采用中文可读命名：`万能投票工具应用.jsx`、`示例数据.js`、`界面样式.css`、`开发日志.md`。

### 3. 已实现功能

- 前台首页：展示已发布任务，支持任务搜索、任务类型和状态展示。
- 任务登录：支持随机账号/指定账号登录校验，并阻止已提交账号重复参与。
- 用户投票页：支持选举和作品投票的多选限额控制。
- 民主测评页：支持多个测评对象共用一套题目，支持评分题和等级评价题。
- 后台任务管理：支持新建、编辑、发布、取消、删除任务。
- 题目与对象配置：支持添加题目、添加候选人/测评对象/作品对象，支持附件名称记录。
- 账号管理：支持按任务生成随机账号、下载账号 CSV。
- 数据统计：支持参与人数、完成人数、完成率、排名、得票数、测评总分、维度均分统计。
- 数据导出：支持导出统计结果 CSV，Excel 可以直接打开。

### 4. 数据结构设计

- `task`：对应报告中的 `vote_task`，保存任务名称、简介、类型、登录方式、匿名设置、状态、时间和创建人。
- `objects`：对应 `vote_object`，保存候选人、测评对象或作品信息。
- `questions`：对应 `vote_question`，保存测评题目、题型、维度、分值和等级选项。
- `accounts`：对应 `task_account`，保存账号、密码、是否已提交。
- `records`：对应 `vote_record / vote_answer`，保存提交账号、提交时间和具体答案。

### 5. 界面设计调整

- 保留报告原型中的蓝色主色和后台侧栏结构，但增强了卡片层次、表格可读性和移动端适配。

- 后台采用工作台式布局，任务管理、配置和统计页面视觉统一。
- 移动端下任务卡片、对象卡片、表单和统计卡片会自动变为单列布局。

### 7. 后续建议

- 接入真实后端：建议使用 Spring Boot / Node.js / Django 任一课程组熟悉的后端，并将 localStorage 数据结构迁移到数据库。
- 增加真实登录与权限：区分平台管理员、后台用户、投票用户。
- 增加文件上传：作品图片和附件当前为名称记录，后续可接对象存储或本地上传接口。
- 增加批量导入：对象和账号可通过 Excel/CSV 批量导入。
- 增加测试用例：建议覆盖防重复提交、任务状态切换、测评维度统计和导出功能。
